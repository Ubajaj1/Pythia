02 · What

What is Pythia?

Describe a decision in plain English. Pythia spawns a panel of agents — each with a persona, a cognitive bias, and relationships to the others — then runs an opinion-dynamics simulation where they argue tick by tick. You watch stances shift in real time.

Agents whose reasoning doesn't match their actions get pulled into a self-correction loop. Their behavioral rules are rewritten. They re-enter smarter.

04 · Architecture

How it actually works

Five layers. One LLM abstraction. Everything observable.

Dashed red = the oracle loop. Incoherent agents cycle through Temple → Engine until all pass (or 5 runs).

The eight mechanisms that matter

  1. Structure enforcement. Every agent tick returns strict JSON — {stance, action, emotion, reasoning, message, influence_target}. Pydantic validates; parse fails fall back to a neutral "confused" tick instead of crashing.
  2. Emotion as feedback. Emotion is not decoration — it's stamped into memory and next tick's prompt. Anxious agents stay anxious. Confident ones double down. Drives momentum and herding.
  3. Bias in two layers. LLM prompts get behavioral cues (text-only biases: framing, availability, authority). Then a mechanical bias function nudges the proposed stance (anchoring pulls toward initial, bandwagon toward aggregate, loss aversion asymmetric). Both leave visible trace.
  4. Parallel ticks. All agents reason simultaneously from last tick's world — not this tick in progress. Kills turn-order bias. Cuts wall-clock by N×.
  5. Memory compression. AgentMemory.for_prompt() keeps the anchor tick, any pivot >0.10 stance delta, plus the last 3 ticks. History stays honest without bloating the context.
  6. Coherence, not accuracy. After a run, evaluator asks per agent: does the stated reasoning explain the action? Flags only direct contradictions, self-contradictions within one tick, or large stance shifts with empty reasoning. Measurable without ground truth.
  7. Temple of Learning. Flagged agents get 1–3 new rules. At 8-rule cap, Temple must edit existing rules instead. Also recommends raise/lower/none on bias_strength. Additive by default — nuance accumulates.
  8. Ensemble for uncertainty. Run N=3 sims with the same agents but fresh tick loops. If 2+ agree → signal. If all 3 disagree → genuine uncertainty. Worst-case confidence when agreement < 60%. This is how we show variability honestly.
05 · Design decisions

Why it's built this way

Coherence over accuracy
No ground truth for "what will people think." Coherence is the strongest honest signal — and backtest mode adds real accuracy scoring when history exists.
Parallel per-tick LLM calls
Agents see last tick's world, not this one. Turn-order bias eliminated; latency cut by N×.
Two-layer bias
Text cues shape reasoning; mechanical functions shape trajectory. Dialogue and dynamics both reflect the bias.
Additive rule amendment
Removing rules makes agents drift generic. Appending keeps every learned nuance; at 8-rule cap, Temple edits instead.
Ensemble = variability
Same agents, same graph, fresh tick loops. Different orderings and LLM sampling surface where the system is stable vs fragile.
Backtest = calibration
Against a known outcome we score direction (±0.1 neutral band), aggregate error, and confidence match. Turns "coherent" into "correct."
Hand-authored bias catalog
16 canonical biases with scientific citation + behavioral cues. No freeform LLM bias names that drift from run to run.
LLMClient Protocol
Structural typing. Any object with generate(prompt, system) → dict plugs in. Token-bucket rate limiter per provider.
SSE end-to-end
UI renders ticks as they stream. No one-shot blocking calls; you watch agents think.
06 · Limitations & future scope

What it can't do yet

Pulled from real run logs — not speculation.

Rate limits throttle demos 40+ × 429 in one oracle run

Free-tier Groq hit back-to-back 429s during parallel ticks. Retry-with-backoff works but stretches 30s runs to 90s.

Next: adaptive concurrency per provider tier.

Coherence ≠ correctness 100% coherent ≠ right

A run can be internally consistent and still wrong about the world.

Next: grow the backtest corpus and calibrate per scenario class.

Small-N panel ~6 agents is the sweet spot

Boardroom-scale works; market-scale blows up prompt + per-tick cost.

Next: named agents + crowd particles driven by aggregate dynamics.

Memory is still a string ~3KB per agent at 20 ticks

Compression works today. Longer horizons need graph summary or vector recall behind the same interface.

Next: swap for_prompt() implementation without touching the engine.

Small LLMs drift Llama 3.1 8B JSON failures

Malformed JSON or verbatim-repeat reasoning still happens; neutral-fallback catches it but it's ugly.

Next: per-archetype model routing — cheap for reactive, precise for analytical.

Static influence graph edges set at gen time

Real opinion networks re-wire under stress. Today's graph stays frozen.

Next: tick-triggered edge updates based on message sentiment.
