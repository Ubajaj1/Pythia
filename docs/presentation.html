04 · Architecture

How it actually works

Five layers, one LLM abstraction, everything observable.

Dashed red is the oracle loop. Incoherent agents cycle through Temple then Engine, up to 5 runs.

The nine mechanisms that matter

  1. Structure is non-negotiable. Every tick returns strict JSON, Pydantic validated, malformed replies fall back to a neutral tick.
  2. Emotion is feedback. Whatever an agent feels this tick gets stamped into memory and replayed next tick.
  3. Bias in two layers. Prompt cues shape reasoning, a mechanical nudge shapes the trajectory.
  4. Guardrails keep runs stable. Stance clamped 0.0 to 1.0, per-tick nudge capped, memory compressed, rules capped at 8, fuzzy targets fall back to null.
  5. Parallel ticks, not turn-taking. All agents reason simultaneously against last tick's world, killing turn-order bias.
  6. Coherence is the honest metric. We score whether reasoning explained the action, not whether the action was right.
  7. Temple rewrites failures. Incoherent agents get new rules or edited rules and a nudged bias strength before the next run.
  8. Ensemble quantifies uncertainty. Three runs agreeing is signal, three disagreeing is reported as low confidence.
  9. Backtest is the accuracy check. Against a known outcome we score direction, aggregate error, and confidence match.
05 · Design decisions

Why it is built this way

Coherence over accuracy
No ground truth for opinion; coherence is the strongest honest signal. Backtest adds accuracy when history exists.
Parallel per-tick LLM calls
Agents see last tick's world, eliminating turn-order bias and cutting latency by N times.
Two-layer bias
Text cues shape reasoning, mechanical functions shape trajectory.
Additive rule amendment
Appending keeps nuance; at the 8-rule cap, Temple edits instead of inflating.
Ensemble for variability
Same agents, fresh tick loops; surfaces where the system is stable vs fragile.
Backtest for calibration
Direction (±0.1 neutral band), aggregate error, confidence match.
Hand-authored bias catalog
16 canonical biases with citations; no freeform LLM bias names that drift.
LLMClient Protocol
Structural typing, per-provider token-bucket rate limiter.
SSE end-to-end
UI renders ticks as they stream, no one-shot blocking calls.
06 · Limitations & future scope

What it can't do yet

Rate limits throttle demos.

Free-tier providers 429 under parallel ticks.

Coherence is not correctness.

Internally consistent can still be wrong.

Small-N panel.

Boardroom-scale works; market-scale blows up prompt size.

Memory is still a string.

Compressed today; longer horizons need graph or vector recall.

Small LLMs drift.

8B-class models sometimes return malformed JSON or repeat prior reasoning.

Static influence graph.

Relationships set at generation; real networks rewire under stress.
