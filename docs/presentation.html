03 · Why

Why Pythia?

Before we even get to LLMs, the real question is: why simulate at all?

1. Why simulations

Every decision has second-order effects you cannot talk your way through in a meeting. Simulation is how engineers stress-test bridges, how epidemiologists model outbreaks, how pilots train for failure modes. The point is to surface the dynamics you would otherwise discover the expensive way, after committing.

2. Why LLMs are good at this

The hard part of human simulation used to be authoring believable personas with internally consistent reasoning. LLMs do that natively. Give one a persona, a bias, a history, and a world state, and it will reason in character across hundreds of ticks. This used to take a research team.

3. What makes Pythia robust

An LLM improvising personas drifts. Pythia pins it down. Every bias comes from a canonical catalog with scientific citations. Every stance passes through a mechanical correction function. Every run is evaluated for coherence, and incoherent agents get rewritten before they re-enter. The LLM is the actor. The engine is the director.

4. Different from asking the best LLM to roleplay

Asking GPT-4 to "simulate six stakeholders" gives you one voice doing six impressions. In Pythia, each agent is its own prompt, its own memory, its own bias, reasoning in parallel against last tick's world. They disagree because they actually have different contexts. Then ensemble mode runs the whole thing three times so you see where the system is stable and where it is fragile.

04 · Architecture

How it actually works

Five layers. One LLM abstraction. Everything observable.

Dashed red is the oracle loop. Incoherent agents cycle through Temple then Engine until all pass, or 5 runs.

The nine mechanisms that matter

  1. Structure is non-negotiable. Every tick, every agent has to return strict JSON with stance, action, emotion, reasoning, message, and influence target. Pydantic checks it. If an LLM hallucinates a malformed response, the engine logs it and substitutes a neutral "confused" tick instead of crashing the whole run.
  2. Emotion is part of the feedback loop. It is not cosmetic. Whatever emotion an agent reports gets stamped into their memory and replayed in the next tick's prompt. Anxious agents stay anxious. Confident ones double down. That is how herd behaviour and momentum actually emerge, the same way they do in a room.
  3. Bias works in two layers. The prompt carries behavioural cues (how a framing-effect person hears a proposal, what an authority-bias person notices). Then after the LLM proposes a stance, a mechanical function nudges it. Anchoring pulls toward the starting position, bandwagon pulls toward the aggregate, loss aversion amplifies downside moves and dampens upside ones. The dialogue shows the bias, and the trajectory shows the bias.
  4. Guardrails keep things from going haywire. This was the biggest risk to solve. Stances are clamped to 0.0 to 1.0 so they cannot run off. Bias coefficients are capped so no single pull can swing a stance more than 0.15 in a tick. Agent memory is compressed to anchor plus pivots plus the last three ticks, so prompts do not bloat as the run gets longer. Rules cap at eight per agent, and after that the Temple has to edit existing ones instead of piling on more. Malformed influence targets fall back to null instead of naming a hallucinated agent. Small mechanisms, but together they keep 20-tick runs stable.
  5. Parallel ticks, not turn-taking. Every agent reasons simultaneously, reading last tick's world state. Nobody gets to react to this tick's in-progress updates, which kills the turn-order bias you get in sequential simulations. It also cuts wall-clock time by N times.
  6. Coherence is the honest metric. We cannot measure "was the agent right about what people will think" without ground truth. So we measure the strongest honest thing: did the stated reasoning actually explain the action taken? The evaluator flags three things: direct contradictions, self-contradictions within a single tick, or large stance shifts with empty reasoning. Everything else is allowed, including changing your mind.
  7. The Temple of Learning rewrites failures. When an agent fails coherence, the Temple adds one to three rules that would have caught the slip. At the eight-rule cap, it has to edit or replace existing rules instead of adding more. It also recommends raising or lowering bias_strength, because sometimes the bias was too weak (ignored) or too strong (the agent became a caricature). The amended agent re-enters the next run carrying the lesson.
  8. Ensemble is how we are honest about uncertainty. One run is an anecdote. Three runs that agree on the verdict and confidence is signal. Three runs that disagree means the scenario is genuinely ambiguous, and we report the worst-case confidence rather than cherry-picking the one we like. This is what "calibrated" actually looks like.
  9. Backtest is the accuracy check. When history gives us a known outcome, we run the simulation blind and score three things: did we get the direction right (with a ±0.1 neutral band), what was the aggregate error, and did our confidence level match how clear-cut the outcome really was. Turns "coherent" into "correct" on the cases where that is possible to measure.
05 · Design decisions

Why it is built this way

Coherence over accuracy
No ground truth for "what will people think." Coherence is the strongest honest signal. Backtest mode adds real accuracy scoring when history exists.
Parallel per-tick LLM calls
Agents see last tick's world, not this one. Turn-order bias eliminated, latency cut by N times.
Two-layer bias
Text cues shape reasoning, mechanical functions shape trajectory. Dialogue and dynamics both reflect the bias.
Additive rule amendment
Removing rules makes agents drift generic. Appending keeps every learned nuance. At the 8-rule cap, Temple edits instead.
Ensemble for variability
Same agents, same graph, fresh tick loops. Different orderings and LLM sampling surface where the system is stable versus fragile.
Backtest for calibration
Against a known outcome we score direction (±0.1 neutral band), aggregate error, and confidence match.
Hand-authored bias catalog
16 canonical biases with scientific citation plus behavioural cues. No freeform LLM bias names that drift from run to run.
LLMClient Protocol
Structural typing. Any object with generate(prompt, system) → dict plugs in. Token-bucket rate limiter per provider.
SSE end-to-end
UI renders ticks as they stream. No blocking one-shot calls, you watch agents think.
06 · Limitations & future scope

What it can't do yet

Rate limits throttle demos

Free-tier providers hit back-to-back 429s during parallel ticks. Retry with backoff works, but stretches a 30-second run to 90.

Coherence is not correctness

A run can be 100% internally consistent and still be wrong about the world. Backtest helps where history exists.

Small-N panel

Six agents is a boardroom, not a market. Scaling past 20 named agents blows up prompt size and per-tick cost.

Memory is still a string

Compressed history works today. Longer horizons will need graph summary or vector recall behind the same interface.

Small LLMs drift

8B-class models sometimes return malformed JSON or repeat prior reasoning. Neutral fallback catches it, but it shows.

Static influence graph

Relationships are set at generation time and do not evolve. Real opinion networks rewire under stress.
