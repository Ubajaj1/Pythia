02 · What

What is Pythia?

You describe a decision in plain English. Pythia generates a cast of agents — each with a persona, a cognitive bias, and relationships to the others — then runs an opinion-dynamics simulation where they argue tick by tick. You watch stances shift in real time.

Agents whose reasoning doesn't match their actions get pulled into a self-correction loop. Their behavioral rules are rewritten. They re-enter smarter.

Plain English in

"Should we raise a Series A?" — no schema, no form, no dropdown hell.

Agents with teeth

16 cognitive biases, influence graph, memory that compounds across ticks.

Self-correcting

An evaluator scores coherence; failing agents get amended and re-run.

03 · Why

Why Pythia?

A single LLM gives you one plausible answer. A panel of 6 biased agents arguing with each other gives you the texture of a decision: where consensus forms, where it fractures, who holds out, and why.

The usual LLM

  • One voice, one answer
  • No disagreement surfaced
  • No mechanism for "wrong and learns"
  • Confidence without calibration

Pythia

  • Many voices with declared biases
  • Influence graph drives disagreement
  • Coherence loop amends failing agents
  • Ensemble runs quantify confidence

04 · Architecture

How it works

Five layers. One LLM abstraction. Everything is observable.

Dashed red path = the oracle loop. Incoherent agents cycle through Temple → Engine until all pass, or 5 runs max.

Parallel ticks

All agents reason simultaneously from last tick's state — no turn-order bias.

Additive amendment

Temple appends behavioral rules, never removes — accumulated nuance survives runs.

Provider-agnostic

Any object with generate(prompt) → dict works. Swap by env var.

05 · Design decisions

Why it's built this way

Every non-obvious choice — and what it buys.

DecisionWhy
Coherence, not accuracy No ground truth for "what will people think." Coherence — did stated reasoning match the action — is measurable and honest.
Parallel per-tick calls Agents see last tick's world, not this one. Eliminates turn-order bias and cuts wall-clock time by N×.
Additive rule amendment Removing rules makes agents drift generic. Appending keeps every learned nuance alive.
Hand-authored bias catalog 16 biases with scientific citation + behavioral cues. LLM freeform would drift — canonical IDs guarantee consistency.
Token-bucket rate limiter Prevents 429s at source. Tier-1-safe defaults (40 RPM Anthropic Haiku) with env-var override.
LLMClient Protocol Structural typing. Any object with generate(prompt, system) → dict plugs in. No inheritance.
SSE streaming end-to-end UI renders ticks live as they arrive. No blocking one-shot API calls.
Custom engine, not OASIS OASIS is social-media focused. Opinion dynamics is simpler, owned, and fits 6-agent boardrooms better than 1000-agent crowds.
06 · Limitations & future scope

What it can't do yet

Pulled from real runs and logs — not speculation.

Rate limits hurt

Free-tier Groq hit 40+ back-to-back 429s during a 6-agent oracle run. Retry-with-backoff works, but stretches a 30s run to 90s. Next: per-provider adaptive concurrency control.

No ground truth

Coherence ≠ correctness. A run can be 100% coherent and wrong. Next: backtest harness against historical events with calibration scoring (already scaffolded).

Small-N panel

6 agents is a boardroom, not a market. Scaling past ~20 named agents blows up the prompt and per-tick cost. Next: hybrid with crowd particles driven by aggregate dynamics.

Memory is a string

AgentMemory.for_prompt() concatenates tick history. 20 ticks ≈ 3KB per agent — fine today, brittle at longer horizons. Next: graph summary or vector recall behind the same interface.

Small LLMs drift

Llama 3.1 8B sometimes returns malformed JSON or repeats the prior tick's reasoning verbatim. Larger models cost 10×. Next: per-archetype model routing — cheap for reactive, precise for analytical.

Static influence graph

Relationships are set at generation time and don't evolve. Real opinion networks re-wire under stress. Next: tick-triggered edge updates based on message sentiment.

Future scope

  • God's Eye View: inject a new variable mid-simulation — engine pauses, re-contextualizes all agents, resumes.
  • Cross-simulation learning: patterns learned in one run seed hypotheses in the next. Lineage graph tracks amendments across domains.
  • Calibrated backtest scores: historical decisions with known outcomes → accuracy curve per scenario class.
  • Per-agent model routing: fast cheap models for reactive agents, precision models for analytical ones.
